# cheatsheet

An always up-to-date personal cheatsheet that maintains itself. Serving as a high-quality context reference for coding agent.

## Always up to date

Cheatsheets go stale the moment you write them — APIs change, your habits
change, and the notes you took six months ago no longer match how you actually
work. **cheatsheet** keeps itself current by continuously learning from your
real code and usage, so what you see always reflects how things work *today*.

## Features

### Seed with a use case

Start by describing a use case — a workflow, a library, a recurring task you
keep looking up. That seed defines what the cheatsheet is *about*.

### Analyze your code on GitHub

From the seed, cheatsheet inspects your actual code on GitHub to ground the
cheatsheet in the patterns you really use — your idioms, your imports, your
conventions — instead of generic boilerplate.

### Learn usage patterns over time

It tracks how your usage evolves: which snippets you reach for, what changes,
what falls out of use. The cheatsheet adapts as your habits and codebase shift.

### Propose new cheatsheets

As patterns emerge, cheatsheet proactively proposes new cheatsheets — surfacing
the things worth capturing before you even think to ask.
