# cheatsheet

A personal, self-maintaining cheatsheet collection — high-quality, current context
references for coding agents, grounded in how *you* actually use a tool.

Each cheatsheet is generated by a Claude Code skill that mines your own GitHub
repos for real usage, grounds it in the tool's current official docs, and writes
a concise sheet whose **Learnings** correct where your usage has gone stale. Two
scheduled GitHub Actions then keep the collection current without you asking.

See [`index.md`](index.md) for the current collection.

## Generate a cheatsheet

Invoke the `cheatsheet` skill in Claude Code with a use case and an explicit target:

```
/cheatsheet outbound HTTP clients --target httpx
```

It resolves your GitHub identity (`gh`), collects the files where you actually use
the target, establishes current best practice (context7 / official docs), and
writes `cheatsheets/<slug>.md` with a reference snippet, your typical usage
patterns, conceptual Learnings, and imperative **Agent rules** a coding agent can
apply directly. The root `index.md` is regenerated automatically.

Private repos are never disclosed: no name, slug, URL, or verbatim content —
patterns learned there are generalized (Constitution P-7).

## Self-maintenance (GitHub Actions)

Two scheduled [`anthropics/claude-code-action`](cheatsheets/self-maintenance-claude-code-action.md)
workflows keep the collection fresh. Both open PRs for review — nothing is
auto-merged.

- **Monthly refresh** — [`.github/workflows/cheatsheet-refresh.yml`](.github/workflows/cheatsheet-refresh.yml)
  re-mines your repos, regenerates every cheatsheet against today's docs, and opens
  **one PR per materially-changed sheet** (a date-only change is ignored; nothing
  changed means no PR).
- **Weekly recommender** — [`.github/workflows/cheatsheet-recommend.yml`](.github/workflows/cheatsheet-recommend.yml)
  scans your last 7 days of GitHub activity, cross-references the existing sheets,
  and opens **one lightweight proposal-stub PR** (`proposals/<slug>.md`) per
  newly-adopted framework not yet covered — including the exact command to generate
  the full sheet once you approve it.

### Setup

Add two repository secrets (Settings → Secrets and variables → Actions):

| Secret | What | How |
|---|---|---|
| `CLAUDE_CODE_OAUTH_TOKEN` | Claude authentication for the action | `claude setup-token` locally |
| `GH_SCAN_TOKEN` | **Read-only** fine-grained PAT to mine your repos | Contents: Read + Metadata: Read, scoped to all your repos |

`GH_SCAN_TOKEN` is read-only and used only for cross-repo reads; PRs in this repo
are opened with the default `GITHUB_TOKEN`. Each workflow fails fast with a clear
message if `GH_SCAN_TOKEN` is missing. The third-party actions are pinned to commit
SHAs (with a trailing `# v1`/`# v6` tag comment) — bump them deliberately.

## How it's built

- Spec-driven: principles in [`specs/CONSTITUTION.md`](specs/CONSTITUTION.md),
  scope in [`specs/INTENT/`](specs/INTENT/) (generation engine = I-1, self-maintenance = I-2).
- Python tooling is `uv`-managed; the only executable code is the deterministic
  index generator [`scripts/register_cheatsheet.py`](scripts/register_cheatsheet.py)
  (`uv run --with pytest pytest` for its tests). All judgment-heavy work lives in
  the skill and the scheduled agents.
