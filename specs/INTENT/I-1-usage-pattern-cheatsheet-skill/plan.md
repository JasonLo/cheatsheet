## Design summary

O-6 is satisfied by a skill-template edit plus a content backfill — no new code, no new deps, no machine-readable artifact (Constitution P-3 simplest-solution, P-6 skills-handle-judgment). The "Agent rules" section is a distinct, imperative ALWAYS/NEVER projection of the same findings that feed the human-oriented Learnings section. The two are deliberately separate altitudes:

- **Learnings** — conceptual old→new mental-model framing for a human reader (O-5).
- **Agent rules** — terse imperative directives a coding agent applies directly without rereading prose (O-6).

Placement: directly **after** Learnings, since the rules are derived from the Learnings (and from Typical usage patterns). This keeps the derivation order readable (concept first, then the distilled directive).

Explicit non-goals carried from the intent (do NOT do these):
- No YAML frontmatter / tags / machine-parseable metadata (cargo-cult for plain markdown; not load-bearing in Claude Code).
- No separate machine file per topic — the rules live inside the single self-contained cheatsheet.
- No auto-wiring into consuming-repo CLAUDE.md / `@`-imports.

## Phase 2 — Backfill the existing cheatsheet

Append an `## Agent rules` section after the existing `## Learnings` block (after line 39). Translate the four Learnings + patterns into imperative directives. Draft content:

```
## Agent rules

- ALWAYS decide tool-vs-glue before choosing the run style: portable tool → `# /// script` + `uv run script.py`; project glue → `uv run python script.py`.
- NEVER insert `python` into `uv run script.py` for a self-contained tool script — it opts out of PEP-723 isolation and uses the project env.
- ALWAYS manage inline deps with `uv add --script` / `uv lock --script`.
- NEVER hand-edit the `# /// script` dependency list — let uv own the metadata.
- ALWAYS strip `VIRTUAL_ENV` before a `uv run` script shells out to child `uv` commands (the inherited value points at the isolated cache env).
- ALWAYS run `uv lock --script` when a self-contained tool needs reproducibility — portability and pinning are independent axes.
```
