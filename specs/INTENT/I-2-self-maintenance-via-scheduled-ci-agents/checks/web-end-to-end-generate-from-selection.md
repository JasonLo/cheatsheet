# Agent check: selecting a recommendation by issue comment generates the sheet and opens a PR — closing the loop entirely in the GitHub web UI

## SHALL
**WHEN** the repository owner comments a topic selection (e.g. "1 and 3") on an open cheatsheet-recommendations issue **THE SYSTEM SHALL** generate each selected cheatsheet via the I-1 skill and open exactly one human-reviewed pull request per selected topic, requiring no local execution — so the recommend → select → generate → review loop completes entirely within the GitHub web interface. [test: agent:specs/INTENT/I-2-self-maintenance-via-scheduled-ci-agents/checks/web-end-to-end-generate-from-selection.md]

## Success criteria
Inspect `.github/workflows/cheatsheet-generate.yml` (and the companion `.github/workflows/cheatsheet-recommend.yml`) and cite file:line evidence for each. PASS requires all of them; FAIL if any is missing or contradicted.

- **Trigger + guard.** `on: issue_comment: types: [created]` is present and paired with `workflow_dispatch`. The job `if:` restricts the comment path to ALL of: (a) the repository owner (`github.event.comment.user.login == github.repository_owner`), (b) an OPEN issue (`github.event.issue.state == 'open'`), (c) a recommendations issue (title-prefix match on `Cheatsheet recommendations`), and (d) not a PR comment (`github.event.issue.pull_request == null`). The `workflow_dispatch` path accepts an issue number + selection so the loop is testable without a live comment.
- **Selection → inputs mapping.** The prompt instructs the agent to read the issue body via `gh issue view`, parse the markdown table, map each selected number to its (use case, target) row, skip numbers with no matching row, and never invent a topic. Untrusted selection text is passed via `env` (`SELECTION_COMMENT`/`DISPATCH_SELECTION`/`ISSUE_NUMBER`), not interpolated into the prompt string.
- **Generation via the I-1 skill.** The prompt directs the agent to follow `.claude/skills/cheatsheet/SKILL.md` to author `cheatsheets/<slug>.md` and to run `uv run scripts/register_cheatsheet.py` to regenerate `index.md`.
- **One human-reviewed PR per topic, no auto-merge.** Per-topic branch (`auto/generate/<slug>`), `git restore`/`git switch` isolation between topics, a dup-PR guard via `gh pr list`, and `gh pr create` per topic. `--allowedTools` includes `gh pr create` + `git push` but EXCLUDES any `gh pr merge`, `gh pr review`, or wildcard `gh pr:*`. `permissions:` grants only `contents: write`, `pull-requests: write`, `id-token: write` (no `issues: write`).
- **Hardening.** Both third-party actions (`actions/checkout`, `anthropics/claude-code-action`) are SHA-pinned (no floating `@vN`); a `concurrency` group and `timeout-minutes` are set.
- **Two-token rule (O-3).** Cross-repo reads are prefixed with `GH_TOKEN=$GH_SCAN_TOKEN`; in-repo writes and the issue read use the default `github.token`; the `GH_SCAN_TOKEN` preflight step is present.
- **Privacy (P-7).** The prompt forbids a private repo's name/slug/URL or verbatim content in any committed file or PR text and requires generalization; public repos may be cited.
- **Web-only loop.** The companion `cheatsheet-recommend.yml` issue table no longer instructs a local `/cheatsheet` command — the only call to action is to comment the number(s) on the issue, so the loop completes entirely in the web UI with no local step.
